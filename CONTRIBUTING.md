# Contributing to Scientific Computing Suite

Thank you for contributing to this project.

## Development Workflow

1. Create a new branch for each feature or fix.
2. Write clean and readable Python code.
3. Add tests for new functionality.
4. Commit changes with clear messages.
5. Push your branch and create a Pull Request.

## Branch Naming

Examples:

- feature/calculator
- feature/matrix
- feature/physics
- bugfix/fix-error

## Commit Messages

Use clear commit messages:

Good:
- Add matrix multiplication function
- Fix division error
- Add physics simulation tests

Bad:
- update
- changes
- fix stuff

## Code Rules

- Follow Python coding standards.
- Add comments where needed.
- Keep functions small and understandable.
- Do not directly push to the main branch.

## Review Process

All major changes should be reviewed before merging into the main branch.
